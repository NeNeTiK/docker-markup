Linux Docker - Guide

Короткий довідник з встановлення, базових команд і найкращих практик для Docker на Linux

Вступ

Docker — система контейнеризації, що дозволяє пакувати додатки з усіма залежностями у легкі ізольовані контейнери. На Linux Docker найчастіше працює нативно й використовує ядро ОС для ізоляції процесів.

Швидка інсталяція (Ubuntu)

Команди нижче показують рекомендований порядок встановлення Docker CE на Ubuntu/Debian:

sudo apt update
sudo apt install -y ca-certificates curl gnupg lsb-release
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o
/usr/share/keyrings/docker-archive-keyring.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] \
https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | \
sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt update
sudo apt install -y docker-ce docker-ce-cli containerd.io

Після інсталяції додайте свого користувача в групу docker, щоб запускати docker без sudo:

sudo usermod -aG docker $USER
newgrp docker
Базові Команди
  • docker run --rm -it alpine sh — запустити тимчасовий контейнер.
  • docker ps — перегляд запущених контейнерів.
  • docker ps -a — всі контейнери.
  • docker images — локальні образи.
  • docker build -t myapp:1.0 . — зібрати образ із Dockerfile.
  • docker rm <id>, docker rmi <image> — видалення.
docker-compose (швидкий приклад) 
version: '3.8'
services:
web:
build: .
ports:
- "8080:80"
volumes:
- ./app:/var/www/html
db:
image: postgres:13
environment:
POSTGRES_PASSWORD: example

docker-compose up -d. Зупинка: docker-compose down.

Найкращі практики
  • Будьте уважні з правами користувачів — не запускайте контейнер з root-доступом без потреби.
  • Оптимізуйте Dockerfile: мінімізуйте шари, використовуйте --no-install-recommends.
  • Використовуйте мультистадійні збірки для зменшення розміру образів.
  • Логування: перенаправляйте логи у зовнішні системи (ELK, Prometheus + Grafana).
  • Скануйте образи на вразливості (Trivy, Clair).
